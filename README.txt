Система управления заказами
Описание
Система управления заказами - это десктопное приложение на Python с графическим интерфейсом для управления заказами, клиентами и товарами. Приложение использует базу данных SQLite для хранения информации.

Функциональность
Основные возможности:
Управление заказами: создание, редактирование, удаление и просмотр заказов

Управление клиентами: добавление, редактирование и удаление клиентов

Управление товарами: управление каталогом товаров с ценами и количеством на складе

Поиск и фильтрация: поиск заказов по клиенту, телефону или адресу доставки, фильтрация по статусу

Автоматические расчеты: автоматический расчет суммы заказа при добавлении товаров

Функции заказов:
Создание новых заказов с указанием клиента, адреса доставки и статуса

Добавление товаров в заказ с указанием количества

Изменение количества товаров в существующем заказе

Автоматический расчет общей суммы заказа

Просмотр истории заказов с детальной информацией

Функции клиентов:
Добавление новых клиентов с контактной информацией

Редактирование данных существующих клиентов

Удаление клиентов (с проверкой на наличие связанных заказов)

Функции товаров:
Управление каталогом товаров

Установка цен и количества на складе

Категоризация товаров

Защита от удаления товаров, которые есть в заказах

Установка и запуск
Требования:
Python 3.6 или выше

Библиотека tkinter (обычно входит в стандартную поставку Python)

Библиотека sqlite3 (входит в стандартную поставку Python)

Запуск приложения:
Убедитесь, что у вас установлен Python

Сохраните код в файл order_management_system.py

Запустите приложение командой:

bash
python order_management_system.py
Структура базы данных
Приложение использует SQLite базу данных orders.db со следующими таблицами:

Таблица customers
id - уникальный идентификатор клиента (INTEGER PRIMARY KEY AUTOINCREMENT)

name - имя клиента (TEXT NOT NULL)

phone - телефон клиента (TEXT)

address - адрес клиента (TEXT)

Таблица products
id - уникальный идентификатор товара (INTEGER PRIMARY KEY AUTOINCREMENT)

name - название товара (TEXT NOT NULL)

price - цена товара (DECIMAL(10,2) NOT NULL)

category - категория товара (TEXT)

stock_quantity - количество на складе (INTEGER DEFAULT 0)

Таблица orders
id - уникальный идентификатор заказа (INTEGER PRIMARY KEY AUTOINCREMENT)

customer_id - ссылка на клиента (INTEGER, FOREIGN KEY)

order_date - дата создания заказа (TIMESTAMP DEFAULT CURRENT_TIMESTAMP)

total_amount - общая сумма заказа (DECIMAL(10,2) DEFAULT 0)

status - статус заказа (TEXT DEFAULT 'В ожидании')

shipping_address - адрес доставки (TEXT)

Таблица order_items
id - уникальный идентификатор позиции (INTEGER PRIMARY KEY AUTOINCREMENT)

order_id - ссылка на заказ (INTEGER, FOREIGN KEY)

product_id - ссылка на товар (INTEGER, FOREIGN KEY)

quantity - количество товара (INTEGER NOT NULL)

unit_price - цена за единицу на момент заказа (DECIMAL(10,2) NOT NULL)

Использование
Работа с заказами:
Нажмите "Добавить заказ" для создания нового заказа

Заполните информацию о клиенте (имя, телефон, адрес)

Выберите статус заказа из выпадающего списка

Добавьте товары через кнопку "Добавить товар"

Для изменения количества товара выберите его и нажмите "Изменить количество"

Сохраните заказ

Управление клиентами:
Нажмите "Клиенты" для открытия управления клиентами

Используйте кнопки "Добавить клиента", "Редактировать" или "Удалить"

При удалении проверяется наличие заказов у клиента

Управление товарами:
Нажмите "Товары" для открытия управления товарами

Используйте кнопки "Добавить товар", "Редактировать" или "Удалить"

Добавляйте новые товары с указанием цены, категории и количества

Редактируйте существующие товары

Удаляйте товары (нельзя удалить товары, которые есть в заказах)

Поиск и фильтрация:
Используйте поле поиска для поиска заказов по имени клиента, телефону или адресу доставки

Используйте выпадающий список статусов для фильтрации заказов

Статусы заказов
В ожидании - заказ создан, но еще не обработан

В обработке - заказ находится в процессе обработки

Отправленный - заказ отправлен клиенту

Доставленный - заказ доставлен клиенту

Отменённый - заказ отменен

Особенности реализации
Автоматическое обновление данных после любых изменений

Валидация вводимых данных (проверка числовых значений)

Защита от удаления связанных данных

Автоматический расчет сумм

Удобный интерфейс с табличным представлением данных

Кнопка "Обновить" для принудительного обновления данных

Технические детали
Графический интерфейс: tkinter с ttk виджетами

База данных: SQLite

Обработка ошибок: комплексная обработка исключений базы данных

Автоматическое создание тестовых данных при первом запуске

Интерфейс
Главное окно содержит:

Панель поиска и фильтрации заказов

Кнопки управления: "Добавить заказ", "Редактировать", "Удалить", "Обновить", "Товары", "Клиенты"

Таблицу с списком заказов

Строку состояния с информацией о количестве загруженных заказов

Поддержка
Приложение автоматически создает базу данных и заполняет ее тестовыми данными при первом запуске. Все изменения автоматически сохраняются в базе данных и отображаются в интерфейсе.
